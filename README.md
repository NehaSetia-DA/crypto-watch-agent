# crypto-watch

A small daily monitor for a handful of coins. It does three things every
morning: pulls prices, scrapes what the crypto press is saying, and writes a
short brief on what moved and what is worth watching. Then a dashboard shows it.

It is built to be picked up. Edit one file, point it at your coins and your
sources, regenerate the scrapers, and it is yours.

## The shape of it

Three kinds of work, kept apart on purpose.

**Build — once, with the skill.** The scrapers are generated, not hand-written.
You run the Zyte scrape skill against each news site and it produces a working
Scrapy spider with web-poet page objects. This is the only step that involves an
agent, and it happens once per site.

**Run — daily, no agent.** Prices come from the CoinGecko API (`fetch_prices.py`),
because a price is a solved problem and scraping a number that has a clean API
would be the wrong tool. The spiders run on a schedule and write the day's
headlines. None of this needs an LLM. It is plain code on a timer.

**Read — daily, one agent.** A Cloud Routine reads the prices and the headlines
and writes the brief (`daily_brief.md` is its prompt). This is the one place an
LLM earns its seat, because the work is judgment, not fetching. It surfaces what
to watch. It does not give investment advice.

## Make it yours

Edit `watchlist.yml`. That is the whole interface.

```yaml
coins:
  - bitcoin
  - ethereum
  - solana
sources:
  - https://www.coindesk.com/
  - https://www.theblock.co/
  - https://decrypt.co/
```

Then generate a scraper for each source. With the Zyte `zyte-web-data` plugin
installed and your `ZYTE_API_KEY` set:

```
/scrape https://www.coindesk.com/ news articles: headline, author, published date, body, url
```

Run it once per source. Each run drops a spider into the Scrapy project. For
news sites this is light work: Zyte API has built-in article extraction, so the
page objects lean on the `article` type rather than hand-rolled selectors. The
skill ends by offering to deploy to Scrapy Cloud — say yes.

## Schedule

- **Prices and spiders:** Scrapy Cloud periodic jobs, daily. Free tier, runs in
  the cloud with no machine of yours awake, keeps job history and logs.
- **The brief:** a daily Cloud Routine, set to run after the jobs finish.

Nothing here depends on your laptop being on, which is what makes it shareable.

## Layout

```
watchlist.yml      the only file you edit
fetch_prices.py    CoinGecko price pull (deterministic, no LLM)
spiders/           generated by the skill, one per source
daily_brief.md     the Cloud Routine prompt (the read)
dashboard.html     opens in a browser, reads data/latest.json
data/latest.json   what the daily pipeline writes; sample committed so the
                   dashboard renders before the first real run
```

## Run it locally

```
pip install pyyaml
python fetch_prices.py
open dashboard.html
```

The dashboard ships with a sample day baked in, so it renders before anything
real has run. Once the pipeline writes `data/latest.json`, that takes over.
